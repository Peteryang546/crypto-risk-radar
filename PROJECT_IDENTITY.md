# Crypto Risk Radar - 项目身份与使命

## 我是谁

**身份**: 国际贸易从业者  
**背景**: 非技术背景，风险厌恶型投资者  
**立场**: 不参与任何非法或未经政府批准的加密货币交易

### 为什么研究区块链

1. **贸易结算防备**: 美元信用体系可能因极端事件受损，小国可能转向区块链币结算
2. **客户需求**: 未来客户可能提出用加密货币结算，需要能够识别风险
3. **信息透明**: 在充满噪声和骗局的市场中，提供可验证的链上事实

## 研究目的

> "在加密货币市场充满信息噪声、操纵和骗局的背景下，通过公开的链上数据和可验证的事实，帮助普通投资者识别高风险代币的典型骗局模式。我们不推荐任何代币，也不预测价格，只提供'项目方说的'与'链上实际发生的'之间的对比，以及历史相似案例的参考。"

### 核心原则

- **只陈列事实，不下结论**
- **不预测涨跌，不提供投资建议**
- **站在坑边举灯的人，不是导师或预言家**
- **模式识别，而非预测未来**: 通过历史案例分析提取可复用的风险模式，形成普通人可用的风控方法论
- **静态观察，动态验证**: 热力图等工具提供静态快照，模式公式用于分析新项目，观察验证而非预测

## 风控方法论

### 目标
建立一套**普通人可用的风控模型**：
1. **模式库**: 从历史案例中提取攻击模式（Rug Pull、Flash Loan、合约漏洞等）
2. **检测公式**: 将模式转化为可操作的检测指标
3. **验证闭环**: 用观察验证分析结果，持续迭代方法论

### 方法
```
历史案例 → 模式提取 → 形成公式 → 分析新项目 → 观察验证 → 迭代优化
```

### 工具
- **静态热力图**: 风险维度的可视化快照（非动态预测）
- **模式检测清单**: 基于历史案例的检查项
- **案例库**: 20个验证案例的模式分析
- **链上事实**: 可验证的交易哈希和合约数据

## 项目使命

1. **避坑指南**: 帮助读者（包括自己）识别骗局，避开陷阱
2. **案例积累**: 为未来可能到来的监管提供真实的案例素材
3. **信息透明**: 让弱者不再被迷雾收割，让坏人无法浑水摸鱼

## 研究风格

- **中立**: 不站队，不推销，不贬低
- **客观**: 用数据说话，用链上事实证明
- **教育性**: 帮助读者理解风险，做出自己的判断

## 免责声明

> 本研究仅供教育和信息参考，不构成任何投资建议。加密货币投资具有极高风险，可能导致全部本金损失。请遵守当地法律法规，在合法合规的前提下进行任何金融活动。
