## 检测模块列表

### 1. AI深度伪造检测

**检测目标**: 项目宣传中是否包含名人视频/直播截图

**检测流程**:
1. 提取项目宣传材料中的图像/视频帧
2. 反向图像搜索（Google Images, TinEye）
3. 检查是否为AI生成（Hive Moderation, Optic AI）
4. 验证名人真实代言记录
5. 关联地址年龄检查（新地址=高风险）

**风险评级**:
- 🔴 高风险: 确认为深度伪造
- 🟡 中风险: 无法验证真实性
- 🟢 低风险: 确认为真实代言

**输出格式**:
```markdown
### AI深度伪造检测
| 项目 | 检测结果 | 证据 | 风险等级 |
|------|----------|------|----------|
| XXX Token | 伪造 | 反向搜索匹配到原视频 | 🔴 高 |
```

### 2. 地址投毒检测

**检测目标**: 检测用户地址与知名地址的相似性

**计算方法**: Levenshtein 距离

**检测流程**:
1. 提取交易对手地址
2. 与知名地址库对比（交易所、名人、合约）
3. 计算Levenshtein距离
4. 距离≤3 → 红色警告

**知名地址库**:
```
# 格式: 地址 | 标签 | 类型
0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb | Binance | 交易所
0x8ba1f109551bD432803012645Hac136c82C3e8C | Vitalik | 个人
```

**风险评级**:
- 🔴 高风险: 距离≤3
- 🟡 中风险: 距离4-5
- 🟢 低风险: 距离>5

### 3. Meme币Rug Pull预判

**检测指标**:

| 指标 | 阈值 | 风险等级 |
|------|------|----------|
| 流动性锁仓比例 | <50% | 🔴 高 |
| Top10持币占比 | >70% | 🔴 高 |
| 创建者历史短命项目 | >3个 | 🔴 高 |
| 合约创建时间 | <7天 | 🟡 中 |
| 社交媒体活跃度 | 无 | 🟡 中 |

**综合评级**:
- 3项及以上🔴 → 极高风险
- 2项🔴 → 高风险
- 1项🔴或2项🟡 → 中风险
- 无🔴且≤1项🟡 → 低风险

### 4. 虚假交易机器人检测

**检测目标**: 合约反编译后门识别

**危险函数签名**:
```solidity
// 自毁函数
function selfdestruct(address payable recipient)

// 任意授权
function approve(address spender, uint256 amount)

// 隐藏铸币
function mint(address to, uint256 amount)

// 交易税可任意调整
function setTax(uint256 newTax)

// 黑名单功能
function blacklist(address account)
```

**检测流程**:
1. 获取合约字节码
2. 反编译为伪代码（Etherscan/在线工具）
3. 模式匹配危险函数
4. 检查代理合约升级权限

**风险评级**:
- 🔴 高风险: 存在selfdestruct/mint/任意approve
- 🟡 中风险: 存在可调整税/黑名单
- 🟢 低风险: 无上述功能

---

### 5. "三无"垃圾代币判定

**判定标准**:

| 条件 | 判定 |
|------|------|
| 无审计报告 | 是/否 |
| 无流动性锁仓 | 是/否 |
| Top10持币>80% | 是/否 |

**判定结果**:
- 3项全中 → 🔴 垃圾代币
- 2项中 → 🟡 高风险
- ≤1项 → 🟢 正常

## 检测执行顺序

```
1. 地址投毒检测（最快，O(n)）
2. "三无"判定（快速，API调用）
3. Meme币Rug预判（中等，多数据源）
4. 虚假机器人检测（较慢，需反编译）
5. AI深度伪造（最慢，需图像分析）
```
